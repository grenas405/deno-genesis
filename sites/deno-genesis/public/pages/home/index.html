System Requirements

  • 🦕 Deno 2.0+ (Runtime of the future)
  • 🗄️ MariaDB 10.6+ (Persistence layer)
  • 🌐 Nginx (Network interface)
  • ⚙️ SystemD (Process orchestration)
  • 🐧 Unix-like OS (Ubuntu 22.04+ recommended)

What Makes DenoGenesis Different

🔧

Unix Philosophy Renaissance

Brings back the elegance of Unix principles with modern capabilities: Do one thing well, composable by design, text-based configuration, direct execution with no build pipelines.

🏢

Real Business Results

Not theoretical - deployed in production with quantified outcomes. 80% cost reduction, sub-100ms response times, 100% uptime during Oklahoma storms. Enterprise performance from individual developers.

🤖

AI-Augmented Development

Framework designed for human-AI collaboration. LLM-friendly architecture, structured documentation, type-safe composition, and explicit permissions that both humans and AI can understand.

🏗️

Hub-and-Spoke Architecture

Centralized framework core with isolated site instances. Shared utilities and patterns, independent deployments, unified monitoring, and zero-conflict multi-tenancy.

🛡️

Security by Design

Explicit permission models, secure defaults, input validation, SQL injection prevention, and comprehensive audit trails. Security that doesn't get in the way.

🎯

Local-First Philosophy

Applications that work offline, sync when available, prioritize user agency, ensure data portability, and maintain business independence from vendor platforms.

The Architecture

// DenoGenesis Hub-and-Spoke Architecture core/ // Centralized framework ├── middleware/ // HTTP request processing ├── database/ // Universal schema + connections ├── utils/ // Shared utilities └── types/ // TypeScript definitions sites/ // Independent applications ├── okdevs/ // Developer community (port 3002) ├── heavenly-roofing/ // Business site (port 3003) └── your-project/ // Your application (port 30XX) // Each site imports core modules import { middleware, db } from "../../core/mod.ts";

This pattern allows shared infrastructure with complete application isolation. Update the framework once, benefit everywhere. Deploy sites independently, monitor centrally.
